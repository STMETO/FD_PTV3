# 📊 联邦模型评估指标设计

## 1. 基础验证指标（已有）
- **Loss**：验证集平均损失，衡量模型在验证集上的拟合程度（越低越好）。  
- **mIoU**：Mean Intersection over Union，逐类 IoU 取平均（语义分割核心指标，越高越好）。  
- **mAcc**：Mean Accuracy，逐类精度取平均，避免被大类别主导。  
- **allAcc**：Overall Accuracy，整体精度，直观但可能受大类别影响。  

---

## 2. 对比指标：联邦模型 vs 本地模型
- **Fed vs Local Accuracy**：全局聚合模型精度对比单客户端本地模型精度。  
- **性能提升率 (Fed Gain)**： 
 
  \[
  \text{Gain} = \frac{\text{FedAcc} - \text{LocalAcc}}{\text{LocalAcc}}
  \]

  衡量联邦模型相对于本地模型的性能提升比例。  
- **差距缩小度**：客户端间性能标准差或极差的变化，反映联邦模型是否让弱客户端受益。  

---

## 3. 公平性与稳健性指标
- **客户端公平性 (Fairness across clients)**：  
  统计每个客户端在本地验证集上的精度，计算方差 / 标准差。数值越小表示越公平。  
- **跨客户端鲁棒性**：  
  在 Client A 的模型上测试 Client B 的数据，观察性能下降情况；与联邦模型对比。  
- **Fairness Gini**：类似基尼系数，描述不同客户端性能的均衡程度。  

---

## 4. 收敛与效率指标
- **收敛速度**：达到相同精度所需的通信轮次数（联邦模型 vs 本地 / 集中训练）。  
- **通信开销 vs 性能**：每轮聚合传输的模型大小，与性能提升的对比。  
- **训练效率**：同等计算资源下，联邦模型和集中式训练的速度比较。  

---

## 5. 可写入 TensorBoard 的建议指标
- `Fed_model_val/loss` → 验证集平均 loss  
- `Fed_model_val/mIoU` → 验证集平均 IoU  
- `Fed_model_val/mAcc` → 验证集平均精度  
- `Fed_model_val/allAcc` → 验证集整体精度  
- `Compare/Fed_vs_Local_Acc` → 联邦模型 vs 本地模型精度对比  
- `Compare/Fed_Gain` → 性能提升率  
- `Compare/Client_Variance` → 客户端性能差异（标准差）  
- `Compare/Fairness_Gini` → 公平性指标  
- `Efficiency/Convergence_Rounds` → 收敛速度  
- `Efficiency/Comm_Cost` → 通信开销  



（2）需要注意的依赖冲突问题
安装过程中，pip 为了满足 FedML 的依赖要求，自动降级 / 替换了部分已安装包，可能影响原有框架（如 PointTransformerV3）的兼容性，关键冲突如下：
冲突包名	原有版本	被降级 / 替换为	潜在影响
numpy	2.2.6	1.26.4	PointTransformerV3 若依赖 numpy 2.x 的新特性（如更优的数组操作），可能出现报错
protobuf	6.31.1	3.20.3	若你的框架中有用到高版本 protobuf 的序列化功能（如模型保存 / 通信），可能不兼容
importlib-metadata	8.7.0	1.7.0	仅影响依赖该包的工具（如yapf代码格式化工具），对联邦学习核心逻辑影响小
wandb	0.22.0	0.13.2	若你之前用 wandb 0.22.0 记录实验，版本降级可能导致部分日志功能失效
